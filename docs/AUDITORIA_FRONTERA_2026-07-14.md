## Síntesis

After the periodic-trig-EQUATION and symbolic-linear-inequality rounds landed, the frontier has moved decisively off the definite/real-domain value core (still holding) and onto three bands. (1) The ALGEBRA of solve with symbolic parameters: inequalities that commit to an undecidable root ordering, abs inequalities with a symbolic center, and radical/inverse-trig equations that drop a partial function's implicit domain/range constraint on the symbolic path while the numeric path applies it. (2) The periodic-family completeness discipline, now recurring in new spellings — reciprocal/power trig EQUATIONS collapsing to a principal representative, plus their reciprocal-square INEQUALITY analogs collapsing to ∅/ℝ/one ray, and nested-abs equations dropping half their roots. (3) A broad eager-evaluation blowup band — symbolic matrix powers, huge integer powers, missing modular exponentiation, cubic-surd rationalization, and repeated differentiation — that hangs or emits multi-MB garbage trees for want of numeric folding, fast algorithms, and cycle-safe orchestration. Two clean capability gaps (Taylor through removable singularities; Leibniz differentiation under the integral sign) and the still-cosmetic complex value domain round it out. Highest leverage is the eager-evaluation blowups (bounded, single-fix, P0 hangs) and the two big solve-completeness clusters (trig-family-drop, nested-abs).

### F3 [P1] taylor-removable-singularity-undefined

- **Representante:** `taylor(sin(x)/x, x, 0, 4)`
- **Correcto:** 1 - x^2/6 + x^4/120
- **Miembros (6):** `taylor(sin(x)/x, x, 0, 4)`; `taylor(x/(exp(x)-1), x, 0, 4)`; `taylor((exp(x)-1)/x, x, 0, 4)`; `taylor((1-cos(x))/x^2, x, 0, 4)`; `taylor(tan(x)/x, x, 0, 4)`; `taylor(sinh(x)/x, x, 0, 4)`

- **Causa-raíz (file:line):** crates/cas_math/src/limits_support.rs:4603
- **Mecanismo:** In the Expr::Div branch of taylor_at_zero_with_rational (limits_support.rs:4600-4609), the call reciprocal_series(&den_series, order, var_name)? unconditionally requires den(0) != 0. For a removable singularity like sin(x)/x, den_series = x has coeffs [0,1] (zero constant term), so reciprocal_series returns None and the Div branch declines. taylor_series_at_zero_expr then returns None, and TaylorRule (cas_engine/src/rules/calculus/taylor.rs:49-58) falls through to the differentiation-from-definition fallback taylor_series_at_point_expr. That path evaluates f^(0)(0) = substitute(sin(x)/x, x->0) = sin(0)/0 = 0/0, which fold_constant_subexprs collapses to the 'undefined' sentinel and leaks as the whole result. All 6 members are Div forms where numerator and denominator both vanish at 0 with equal valuation, so all hit this same decline-then-leak path.
- **Boceto de fix:** In the Expr::Div branch, keep the existing path as an early-return when reciprocal_series succeeds (identical behavior for non-singular quotients, vd=0). When it fails (den(0)=0), add a removable-singularity fallback: (1) compute denominator valuation vd = den_series.coeffs.iter().position(|c| !c.is_zero()) — None means den is identically zero, decline; (2) re-expand num and den to order+vd (needed so the quotient still has order+1 coefficients after cancelling x^vd — e.g. sin needs order 5 to recover x^4/120); (3) cancel the common x^vd via a shift_down helper that drops vd leading coeffs from each, returning None if any of the numerator's first vd coeffs is non-zero (numerator valuation < vd = genuine pole, honestly decline); (4) the shifted denominator now has a non-zero constant term, so reciprocal_series succeeds — multiply by the shifted numerator series and truncate_polynomial to order. Two small local helpers (valuation, shift_down); reuses reciprocal_series, truncate_polynomial, Polynomial::{new,mul,coeffs,is_zero}. Entirely within the one Div match arm.
- **Blast radius:** Low. taylor_at_zero_with_rational is a private fn reached only via the public taylor_series_at_zero_expr, which is called only by TaylorRule (the taylor()/series() command at expansion point 0). The limit engine's series path uses the SEPARATE taylor_at_zero (not the _with_rational variant, by design comment at limits_support.rs:4585-4589) and is untouched. The non-singular quotient path is preserved byte-for-byte via early-return. Genuine poles keep declining because numerator valuation < denominator valuation makes shift_down fail: verified 1/x, 1/x^2, (1+x)/x, cos(x)/x, 1/(exp(x)-1), (sin(x)-x)/x^5 all currently return 'undefined' and stay declined. Existing tests rational_taylor_declines_pole_at_zero (10830) and rational_taylor_matches_known_geometric_series (10820) remain green.
- **Maquinaria reutilizable:** reciprocal_series (limits_support.rs:4569, already implements the 1/den power-series recurrence and requires den(0)!=0 — satisfied after cancellation), truncate_polynomial (4817), Polynomial::new/mul/coeffs/is_zero (polynomial.rs). Polynomial::new already trims trailing zeros so valuation (lowest non-zero index) is well-defined. No new algorithms needed.
- **ROI:** The Taylor expander evaluates the operand at the expansion center, hits 0/0, and propagates the operand's NonZero domain condition into a leaked 'undefined' sentinel instead of expanding through the removable singularity. One self-contained series-engine fix — expand numerator and denominator series and divide (or take the limit) when the center is a removable singularity — closes all 6. No solver coupling; clean bounded capability. High ROI.
